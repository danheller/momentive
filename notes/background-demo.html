Block backgrounds & accent colors

Two independent systems, meant to be combined: style classes decide which shape renders (dots, ellipse mask, or both), and the --page-accent-color variable decides what color it renders in. Set the color once per page and every block below it can reference it — no per-block wiring.

Simplified for this demo: each block below is just a labeled rounded rectangle. Swap the label content for real headings/paragraphs/images in production.

On solution/product pages, --page-accent-color is set once on <body> from the post's ACF accent-color field. --page-accent-color-bg and --page-accent-color-bg-soft are derived from it with color-mix(), so re-theming a page is a one-variable change — nothing per-block. The ellipse mask SVG (assets/images/Ellipse-Base-Gray.svg) is inlined above as a data URI so this file has no external dependencies; swap it for a url() path if you'd rather ship the file alongside your CSS.
